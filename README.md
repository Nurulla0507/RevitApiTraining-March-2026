# Revit API Training - March 2026

## 📘 Overview

This repository is the central resource hub for the **Revit API Training (March 2026)** cohort.

- Instructor content, sample scripts, and resources are shared here.
- Students clone this repository and add their exercise solutions for review.
- Everyone can refer to this repo for course updates and final reference materials.

## 🧩 Goals

1. Teach Revit API fundamentals via Python scripting.
2. Provide a standard repo structure for student exercises and instructor examples.
3. Enable easy review of student submissions and collaborative learning.

## 📁 Repo Structure (Recommended)

- `samples/` - instructor demo scripts with comments.
- `exercises/` - student submissions, one folder per student or per exercise.
- `resources/` - training slides, API docs links, Revit families, templates.
- `README.md` - this overview.

## 🚀 Get Started (Student Workflow)

1. Clone repo:
   - `git clone https://github.com/akryazbim/RevitApiTraining-March-2026.git`
2. Create your own branch:
   - `git checkout -b student-<yourname>`
3. Add your exercise folder:
   - `exercises/<yourname>/exercise-01/` etc.
4. Commit and push:
   - `git add .`
   - `git commit -m "<yourname> exercise 01 submission"`
   - `git push origin student-<yourname>`
5. Open a Pull Request for review.

## 🧪 Exercise Submission Format (Suggested)

- `exercise-x.py` - Python code.
- `README.md` - explanation of approach, dependencies, and test instructions.
- `results/` - screenshots, output logs, and short notes.

## 🧰 Environment Setup

- Revit version (as specified by instructor)
- Python (either 3.7+ or matching Revit Python environment)
- `pythonnet` / `RevitAPI` references as required by script.

## 🙌 Best Practices

- Keep code modular and nicely commented.
- Commit frequently with clear messages.
- Use explicit names for scripts and paths.
- Push early, iterate often.

## 👥 Review Process

- Instructors/mentors will comment PRs with feedback.
- Students update submissions and re-request review.
- Peer review is highly encouraged.
